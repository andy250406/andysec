# ‘클릭픽스’ 공격으로 700여개 사이트 해킹... 클로드 발견 ‘Ghost CMS’ 취약점 악용

## 📌 핵심 요약

최근 오픈소스 콘텐츠 관리 시스템(CMS)인 Ghost CMS에서 발견된 심각한 SQL 인젝션 취약점(CVE-2026-26980)을 악용한 대규모 '클릭픽스(ClickFix)' 공격으로 700여 개 이상의 웹사이트가 해킹당했다.

공격자들은 패치되지 않은 사이트의 관리자 API 키를 탈취하여 악성 자바스크립트 로더를 삽입, 가짜 Cloudflare 인증 페이지를 통해 사용자에게 악성코드를 설치하도록 유도했다.

## 🔍 주요 내용

* Ghost CMS의 Content API에서 발견된 SQL 인젝션 취약점(CVE-2026-26980, CVSS 9.4)이 공격에 악용되었으며, 이는 인증되지 않은 공격자가 데이터베이스에서 임의의 데이터를 읽을 수 있게 허용한다.
* '클릭픽스' 공격은 2026년 5월 7일에 처음 포착되었으며, 하버드, 옥스포드 등 유명 대학 및 기관을 포함한 700개 이상의 사이트가 영향을 받았다.
* 공격자들은 탈취한 관리자 API 키를 이용해 게시물에 악성 자바스크립트 로더를 삽입하고, 가짜 Cloudflare CAPTCHA 페이지를 통해 사용자에게 원격 접근 트로이목마(RAT) 및 정보 탈취 악성코드(인포스틸러)를 설치하도록 유도했다.
* 해당 취약점은 2026년 2월 19일에 공개되었음에도 불구하고 많은 사용자가 제때 패치하지 않아 공격에 노출되었다.
* 클릭픽스 공격은 가짜 브라우저 업데이트 알림이나 오류 메시지를 통해 사용자를 속여 악성 스크립트를 실행하게 하는 사회공학적 기법을 활용한다.
## 패치 지연과 공급망 공격의 위험성

이번 '클릭픽스' 공격은 널리 사용되는 오픈소스 CMS의 알려진 취약점에 대한 패치 지연이 얼마나 큰 피해로 이어질 수 있는지를 명확히 보여준다. 공격자들은 이미 공개된 취약점을 악용하여 대규모 웹사이트를 침해하고, 이를 통해 최종 사용자에게까지 악성코드를 유포하는 공급망 공격 형태로 진화하고 있다. 이는 단순히 개별 시스템의 보안을 넘어, 소프트웨어 생태계 전반의 신속한 보안 업데이트와 관리의 중요성을 강조한다.

---



> ***알려진 취약점이라도 패치 적용이 늦어지면 대규모 피해로 이어질 수 있으므로, 보안 업데이트는 최우선 순위로 즉시 적용해야 하며, 공급망 전반의 보안 관리 체계를 강화해야 한다.***
