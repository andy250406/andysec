# 구글 클라우드 API 키 남용을 통한 '제미나이(Gemini)' 비공개 데이터 노출 위험

## 📌 핵심 요약

구글 클라우드 환경에서 기존에 발급된 API 키들이 제미나이(Gemini) API 활성화 시 예기치 않게 인증 권한을 동시 획득하여 비공개 데이터가 노출될 수 있는 심각한 취약점이 발견되었습니다.

공격자는 소스코드 등에 노출된 API 키를 악용하여 사용자의 캐시, 저장된 AI 컨텍스트 등 비공개 데이터에 무단 접근할 수 있으며, 과도한 API 호출로 인해 천문학적인 요금이 부과되거나 정상적인 서비스 운영이 중단될 수 있습니다.

## 🔍 주요 내용

* 보안 연구기관의 조사 결과 약 2,800~3,000개 이상의 공개된 구글 API 키가 해당 취약점에 노출되었으며, 여기에는 다수의 주요 금융 기관 및 글로벌 기업도 포함된 것으로 확인되었습니다.
* 과거 구글맵스처럼 클라이언트 측 웹사이트 코드에 단순 청구 식별 목적으로 노출되어 있던 API 키가, 동일 프로젝트 내 제미나이 활성화로 인해 권한 에스컬레이션을 일으킨 것이 주된 원인입니다.
* 구글은 유출이 확인된 키들의 제미나이 접근 차단 및 새로운 AI 스튜디오 키의 전용 범위 제한 등 완화 조치를 도입했지만, 기존 사용자들 전체에 대한 소급 조치와 대응에는 여전히 주의가 필요한 상황입니다.
## 단일 서비스 접근 권한 확장이 불러온 AI 인프라의 연쇄 보안 위협

과거 클라이언트 환경에 노출되어도 큰 문제가 없던 단순 빌링용 API 키가 최신 AI 서비스 생태계와 결합하면서 막대한 재무적 손실(요금 폭탄) 및 비공개 데이터 유출 경로로 돌변한 대표적 사례입니다. 레거시 시스템과 신규 AI 엔드포인트를 급하게 통합하는 과정에서 '단일 서비스 권한 에스컬레이션(single-service privilege escalation)' 이슈가 발생했습니다. 이를 방지하기 위해서는 프로젝트 내 API 키들의 스코프(Scope)를 제한하고, 소스 코드 위주가 아닌 Secrets Manager 같은 안전한 저장소로 관리 체계를 즉각 전환하는 것이 필수적입니다.

---



> ***AI 서비스 도입의 속도전 이면에는 레거시 인증 체계와의 충돌이라는 거대한 사각지대가 숨어 있다. API 키 하나가 기업의 핵심 데이터 자산과 막대한 재무를 위협할 수 있음을 인지하고, 즉각적인 서비스별 권한 최소화 및 제로 트러스트 기반의 키 생애주기 강화가 시급하다.***
